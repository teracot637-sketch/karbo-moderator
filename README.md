# karbo-moderator

Модератор-бот для KarboAI. На питоне, через офиц. SDK `karbo`.

В целом всё что умеет это выдавать варны, кикать по лимиту, и автоматически кикать за 18+ картинки (через nudenet).

## Команды

- `/warn [причина]` - выдать варн. Реплаем или с упоминанием (`@ник` или uuid). Когда варнов накопилось столько же сколько лимит - юзера кикает.
- `/unwarn` - снять последний варн.
- `/kick [причина]` - просто кикнуть.
- `/warns` - посмотреть сколько у тебя (или у того кому реплай) варнов и nsfw-страйков.
- `/setwarns N` - поменять лимит варнов в этом чате (только организатор). От 1 до 100.
- `/setnsfw N | on | off` - поменять лимит NSFW-страйков, или включить/выключить авто-модерацию 18+. Тоже только организатор.
- `/leave` - бот уходит из чата. Может звать организатор чата либо владелец бота (по `BOT_OWNER_ID`).
- `/help` - список команд.

`/warn`, `/unwarn`, `/kick` и авто-NSFW работают только если у бота есть роль помощника в чате. Иначе он молча игнорит.

## Запуск

```
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# открой .env и впиши хотя бы KARBO_BOT_TOKEN и BOT_OWNER_ID если хочешь иметь возможность удалять бота из чатов.
python bot.py
```

Первый запуск может занять секунд 30 - nudenet потащит ONNX-модель.

## .env

Минимум что нужно:
- `KARBO_BOT_TOKEN` - токен бота из дев-панели в магазине Karbo, обязательный.
- `BOT_OWNER_ID` - твой uuid - достать должен сам с этим помогать не буду сорян, чтобы /leave можно было дёрнуть из любого чата и бот вышел.

Остальное опционально, дефолты норм:
- `DB_PATH` - путь к sqlite базе, по дефолту `moderator.db`.
- `DEFAULT_WARN_LIMIT` - лимит варнов в новом чате (10).
- `NSFW_ENABLED` - 0 чтобы вырубить авто-модерацию 18+ полностью.
- `NSFW_THRESHOLD` - 0..1, порог уверенности детектора (0.6).
- `NSFW_WARN_LIMIT` - сколько 18+ страйков до кика (3). Считается отдельно от обычных варнов.
- `HELPER_ROLE_MIN`, `ORGANIZER_ROLE_MIN` - значения `role` в Karbo для помощника/организатора (1 и 2).
- `BOT_ROLE_TTL` - на сколько секунд кэшировать роль самого бота, чтобы не дёргать API (300).
- `CMD_PREFIX` - префикс команд, по дефолту `/`. Можно поменять на `!` или что хочешь.
- `LOG_LEVEL` - DEBUG / INFO / WARNING / ERROR.

## Что важно знать

API Karbo не даёт удалять сообщения, мутить или банить. Можно только кикнуть. Все остальные ограничения (лимиты, предупреждения и тд) - это уже логика самого бота.

Варны и страйки лежат в локальной sqlite-базе. Сбрасываются автоматически когда юзера кикают (или вручную через sql, если надо).

NSFW-проверка использует [nudenet](https://github.com/notAI-tech/NudeNet), реагируем на классы `*_GENITALIA_EXPOSED`, `*_BREAST_EXPOSED`, `BUTTOCKS_EXPOSED`, `ANUS_EXPOSED`.

`/warn`, `/unwarn`, `/kick` работают и по реплаю и по упоминанию (`@ник` или просто uuid в тексте).
